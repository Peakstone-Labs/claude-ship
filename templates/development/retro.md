# {FeatureName} - Retrospective

## Metrics Overview
时间、文件数、测试数、发现缺陷数、loop 轮数、决策偏差数。

## Design vs Implementation Gap
功能偏差和实现细节偏差，归因和影响。

## Issues Classification
按根因分类 review 和 qa 的所有发现：
- 盲点（根本没想到）
- 遗漏（想到了但没做）
- 规则缺失（流程/规范未覆盖）
- 外部约束（不可抗力）

## Process Efficiency
本 feature 流程成效、相比之前的进步、可改进点。

## Generalizable Lessons
从本 feature 提炼的可复用经验（候选 memory，需过三条准入）。

## Memory Candidates（三条准入过滤后）

通过 **Non-Googleable + Codebase-Specific + Hard-Won** 三条的条目。每条：

- 规则
- **Why:** （为什么重要 —— 过去的事故或强偏好）
- **How to apply:** （何时/何处应用）

未过三条 → 只写入 Generalizable Lessons，不建议存 memory。

## Process Documentation Updates
对 CLAUDE.md 或流程文档的更新建议。

## Forward-Looking Warnings
对下游 feature 的前瞻提醒和依赖说明。

## Conclusion
质量评价和核心成功因素。
